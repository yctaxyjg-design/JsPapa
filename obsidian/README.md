# 🗂️ 옵시디언 볼트 정리기 (`obsidian/`)

옵시디언 볼트(마크다운 폴더)를 **브라우저에서 열어** 태그·백링크·고아 노트·깨진 링크를
한눈에 살펴보고, 볼트에 다시 넣을 **정리 리포트(.md)** 를 만드는 정적 웹앱입니다.

> 브런치 글 *"Claude code로 Obsidian을 관리하는 이유"* 에서 본 "볼트를 정리해 주는" 작업의
> 가벼운 브라우저판입니다. 로컬에 Claude Code를 붙이기 전에, 혹은 아이패드에서 바로 쓸 수 있어요.

## 하는 일

- **볼트 열기** — 데스크톱 브라우저는 폴더 통째로(File System Access API), 아이패드 사파리는
  파일 앱에서 `.md` 여러 개를 골라서. 크롬 계열은 폴더 선택(`webkitdirectory`)도 지원.
- **한눈에 보기** — 노트 수, 총/평균 글자, 태그 종류, 고아 노트, 깨진 링크 통계 타일.
- **태그** — `#인라인태그` 와 프론트매터 `tags:` 를 모아 많이 쓴 순으로. 눌러서 노트 필터링.
- **노트 목록** — 제목·본문 검색, 고아/깨진 링크 필터, 백링크·나가는 링크·태그 개수 배지.
  노트를 누르면 가벼운 마크다운 미리보기(위키링크·태그 강조).
- **정리 리포트** — 통계·태그 목록·고아 노트·깨진 링크를 담은 마크다운을 만들어
  복사하거나 `정리-리포트.md` 로 내려받아 볼트에 넣으면 지도(MOC)처럼 씁니다.

## 프라이버시

선택한 노트는 **브라우저 안에서만** 읽고 분석합니다. 어디로도 업로드하지 않고,
네트워크 요청이 전혀 없습니다. 탭을 닫으면 아무것도 남지 않습니다.

## 실행

빌드 도구·외부 의존성이 없는 정적 사이트입니다.

```sh
cd obsidian
python3 -m http.server 8000
# → http://localhost:8000
```

폴더 열기(File System Access API)는 보안 정책상 `https://` 또는 `localhost` 에서만 동작합니다.
GitHub Pages 등에 올리면 아이패드에서도 파일 선택으로 바로 쓸 수 있습니다.

## 인식/분석 로직 자가 테스트

파싱·분석은 DOM 없는 순수 모듈 `vault.js` 에 있어 Node에서 바로 검증됩니다.

```sh
node obsidian/vault.test.js
# 프론트매터/인라인 태그, 위키링크 정규화, 백링크·고아·깨진 링크,
# 리포트 생성이 맞는지 확인 → "33 passed, 0 failed"
```

## 파일 구성

- `index.html` — 진입점
- `vault.js` — 볼트 파싱·분석(태그·링크·백링크·고아·리포트). 외부 의존성 없음, Node에서도 로드 가능
- `vault.test.js` — 자가 테스트 (Node 실행)
- `app.js` — 화면 연결부(볼트 읽기, 렌더, 검색·필터, 리포트 복사/내려받기, 미리보기 모달)
- `styles.css` — 애플 HIG 토큰 재사용 + 다크모드

## 한계

- **폴더 열기**는 크로미엄·데스크톱 사파리 등 File System Access API 지원 브라우저에서만 됩니다.
  아이패드 사파리는 폴더 API가 없어 파일 앱에서 `.md` 를 여러 개 골라 여세요.
- 링크 해석은 파일 이름(basename) 기준입니다. 서로 다른 폴더에 같은 이름 노트가 있으면
  첫 번째 노트로 이어집니다(옵시디언의 정확한 최단경로 규칙까지는 흉내 내지 않습니다).
